# AI编码助手实战指南

AI 编码助手的价值不在“自动生成代码”，而在“缩短从想法到可运行结果的路径”。

## 适合交给 AI 的任务

- 脚手架代码和重复模板
- 单元测试草稿与边界用例补全
- 接口类型定义、数据转换函数
- 复杂报错的根因排查建议

## 不适合完全交给 AI 的任务

- 架构决策与核心业务约束
- 涉及安全边界的实现
- 高耦合旧代码的大规模重构

## 高质量提示词模板

你可以直接复用下面这个模板：

```txt
你是资深工程师，请在不改变现有架构的前提下完成修改：
1) 目标：
2) 当前约束：
3) 需要改动的文件范围：
4) 输出要求：给出可直接粘贴的代码与改动说明
5) 验证方式：需要通过 lint/typecheck/tests
```

## 实战工作流

1. 先让 AI 解释现有代码，再要求修改
2. 每次只给一个清晰目标，避免“全量改造”
3. 要求输出“最小可行改动”
4. 要求补充验证步骤并执行

## 质量把关清单

- 代码风格是否符合仓库规范
- 是否引入了未安装依赖
- 是否破坏已有边界条件
- 是否有可回归的测试

## 经验结论

把 AI 当作“高效结对编程伙伴”，而不是“自动替代者”，产出的稳定性会明显更高。
